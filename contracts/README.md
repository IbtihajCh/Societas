# Contracts

**Owner:** All Developers

## Purpose

Defines the API contracts, data schemas, and examples for the SOCIETAS simulation. Ensures consistency across frontend, backend, and simulation modules.

## Structure

```
contracts/
├── api/
│   └── openapi.yaml          # OpenAPI 3.0 specification
├── schemas/
│   ├── agent_state.json      # Agent state schema
│   ├── simulation_state.json # World state schema
│   ├── policy.json           # Policy schema
│   ├── decision_request.json # AI decision request schema
│   ├── decision_response.json# AI decision response schema
│   └── news_event.json       # News event schema
└── examples/
    ├── agent_state_example.json
    ├── simulation_state_example.json
    └── policy_example.json
```

## API Specification

The OpenAPI specification (`api/openapi.yaml`) defines all REST endpoints:

- `/health` - Health check
- `/simulation/*` - Simulation control (status, start, stop, tick, state, reset)
- `/policies/*` - Policy management (list, create, get, revoke)
- `/metrics/*` - Metrics and dashboard data
- `/agents/*` - Agent browsing and details

## JSON Schemas

All data structures are validated against JSON schemas in `schemas/`:

- **agent_state.json** - Complete agent state with traits, needs, emotions, resources
- **simulation_state.json** - World state with economy, crime, needs, psychology
- **policy.json** - Government policy definition
- **decision_request.json** - Request format for AI decision making
- **decision_response.json** - AI decision response format
- **news_event.json** - News event generated by simulation

## Examples

Example data in `examples/` demonstrates valid instances of each schema.

## Usage

### Backend

Import schemas for validation:

```python
import json
from jsonschema import validate

with open('contracts/schemas/agent_state.json') as f:
    schema = json.load(f)

validate(instance=agent_data, schema=schema)
```

### Frontend

Generate TypeScript types from schemas:

```bash
npx json-schema-to-typescript -i contracts/schemas/*.json -o src/types/contracts/
```

### Testing

Use examples as test fixtures:

```python
with open('contracts/examples/agent_state_example.json') as f:
    example = json.load(f)
```

## Versioning

All schemas are versioned. Breaking changes require:

1. Increment major version in schema `$id`
2. Update OpenAPI spec version
3. Update CHANGELOG.md
4. Provide migration guide
