# SOCIETAS Glossary

Common terminology used throughout the project.

---

## A

**Agent**
An autonomous individual in the simulation. Each agent has psychological traits, economic status, needs, and decision-making capability.

**Agent Based Modeling (ABM)**
A simulation paradigm where individual agents interact based on rules, producing emergent system-level behavior.

**Ambiguity Threshold**
The score difference below which the deterministic engine considers a decision ambiguous and escalates to Gemma. Default: 0.05.

## D

**Decision Pipeline**
The sequence of computations an agent follows each tick: evaluate needs → compute action scores → check ambiguity → (optionally) escalate → select action → execute.

**Deterministic**
A system is deterministic if the same input always produces the same output. The core simulation engine is fully deterministic.

**Dual-Process Architecture**
Cognitive model with System 1 (fast, deterministic) and System 2 (slow, reasoning) borrowed from cognitive psychology.

## E

**Escalation**
The process of sending an ambiguous decision to Gemma for reasoning assistance.

**Explainability**
The property that every decision can be traced to its inputs and intermediate computations.

## F

**Fusion (Hybrid Decision Fusion)**
The process of blending deterministic scores with Gemma's scores using configurable weights.

## G

**Gemma**
Google's open-source LLM family. Used as the cognitive reasoning layer (System 2). Models: Gemma 2 9B, Gemma 2 26B.

## L

**Layer 1 — Deterministic Simulation**
The core engine. Fully deterministic. No LLM involvement. Handles world state, agents, economy, needs, psychology.

**Layer 2 — Cognitive Reasoning**
The Gemma integration layer. Handles tie-breaking, policy translation, narrative generation.

**Layer 3 — Presentation**
The user interface layer. Dashboard, charts, news feed.

## P

**Persona**
A one-time generated identity for each agent. Two sentences describing their background and worldview. Generated by Gemma, stored permanently.

**Policy**
A configurable rule set that modifies how agents behave. Policies influence needs weights, decision scores, and economic parameters.

## S

**System 1**
Fast, automatic, deterministic decision-making. Handled by Layer 1.

**System 2**
Slow, deliberate, reasoning-based decision-making. Handled by Layer 2 (Gemma).

## T

**Tick**
A single time step in the simulation. During each tick, every agent evaluates its state, makes a decision, and executes an action.

## V

**vLLM**
High-throughput LLM serving framework. Used to serve Gemma models for inference.
